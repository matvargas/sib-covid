TRASNFORMAÇÕES DE DADOS

Como passamos de arquivos .csv para os gráficos apresentados

Antes de utilizarmos os dados para a confecção dos gráficos disponibilizados foi necessário realizar o procedimento de ETL para garantir que os mesmos estivessem limpos e consistentes. Esse procedimento foi dividido em cinco fases, sendo elas: leitura dos dados, remoção de colunas majoritariamente nulas, remoção de registros com muitos campos nulos, formatação dos campos de datas e carregamento dos dados.

Na primeira fase de leitura dos dados o dataset foi baixado do sistema e-SUS Notifica e carregado dentro do script de ETL. Logo em seguida, após realizar uma análise quantitativa da base removemos uma série de colunas devido ao baixo número de dados preenchidos nas mesmas. Após a normalização das colunas também eliminamos linhas que continham poucas informações, isto é, com muitos campos faltantes ou com campos cruciais definidos como nulos. O último procedimento antes do carregamento dos dados foi a formatação dos campos de datas para um padrão que seria reconhecido pelo nosso sistema de bancos de dados (MySQL). Por fim, após toda a limpeza e normalização da base os dados foram carregados ao banco.

Caso queira saber sobre o processo de ETL, ou do projeto como um todo o código está disponível em um repositório no GitHub.
